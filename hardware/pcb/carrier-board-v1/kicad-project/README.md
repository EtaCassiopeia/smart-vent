# KiCad project — placement-verified, NOT routed, NOT fab-ready

`smart_vent_v1.kicad_pcb` was generated by `build_board.py` (run against
KiCad 10.0.4's bundled `pcbnew` Python module — see the script for how to
invoke it; it bypasses the GUI-only netlist-import step, which modern KiCad
no longer exposes to scripting). It loads real footprints from KiCad's stock
libraries where one exists, places everything at the `place_smart_vent.py`
floorplan coordinates, and wires nets per `../smart_vent_board.py`'s
connectivity. **No copper has been routed.**

`drc_report.json` is the DRC run against this placement. It surfaces real,
useful findings beyond the expected "unrouted" warnings:

## Footprints with no exact stock match (placeholders, need sourcing)

| Ref | Part | Stand-in used | What's actually needed |
|---|---|---|---|
| U1 | Seeed XIAO ESP32-C6 | generic 1×7 pin header | The official Seeed XIAO footprint (KiCad's stock libraries only ship `MCU_Seeed_ESP32C3`, not the XIAO carrier module). Source from Seeed's own KiCad library or via `easyeda2kicad` against the LCSC part. |
| J1 | "SM-2P" battery connector | generic 1×2 pin header | The hobby-RC "JST-SM" 2-pin plug is not a real JST series (despite the name) and isn't in KiCad's `Connector_JST.pretty`. Needs a footprint sourced from the connector's actual datasheet/LCSC listing, or substitute a connector that *is* in-library (e.g. a real `JST_PH` 2-pin) and adjust the BOM. |
| C2 | 470 µF / 10 V electrolytic | `CP_Radial_D8.0mm_P3.50mm` (THT) | The SMD `CP_Elec_6.3x5.4` footprint the original netlist specified isn't in this KiCad install's stock library — confirm it exists in a newer/different library, or use a THT radial cap (changes board-side clearance near J2). |

## Real placement problems found by DRC (not just silkscreen cosmetics)

At actual footprint sizes, the floorplan from `smart_vent_floorplan.svg` is
too tight in several spots — this DRC run found genuine copper-to-copper
shorts, not just overlapping silkscreen text:

- **Shorts** (`shorting_items`, severity error): C2 GND pad ↔ C3 SERVO_VCC
  pad; D1 GND pad ↔ C5 V3V3 pad; D1 LED_DIN pad ↔ C5 GND pad.
- **Clearance violation**: R1 (VBAT) ↔ R2 (NFET_GATE).
- **Courtyard overlaps** (would collide if placed/soldered as-is): R1↔R2,
  SW1↔J1, Q1↔Q2, R7↔D1, D1↔C2, D1↔C5, R3↔C2, C2↔C3.

The cluster around D1/C2/C3/C5/R7 (bottom-right, the LED + servo-bulk-cap
area) is the worst offender — those five parts need real breathing room.
99 `silk_overlap` / 59 `silk_over_copper` / 12 `silk_edge_clearance`
warnings are cosmetic (reference-designator text placement) and not
included in the count above.

## What this project is useful for

Open it in KiCad's PCB editor to skip straight to: nudge the parts listed
above apart (the floorplan's *zones* — left=signal, right=power,
bottom=connectors — are still the right idea; only the exact coordinates
need spreading out), swap in real footprints for U1/J1/C2 once sourced,
then route per the five rules in
[`../../../../docs/battery-carrier-board.md`](../../../../docs/battery-carrier-board.md#5-board-layout-rules),
run DRC again, and plot Gerbers.

Gerber/drill export was smoke-tested against this same unrouted board (to
validate the fab-settings pipeline, not the board itself) and produces the
expected JLCPCB-compatible layer set — that part of the doc's §6 is
confirmed accurate.
